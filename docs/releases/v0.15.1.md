# Release Notes — SentinelAI v0.15.1

## 📌 Release Name: GitHub Presentation & Phase 15A Stabilized Release

This release establishes the core platform and interactive documentation setup for SentinelAI. It introduces the upgraded AI Security Copilot Workspace alongside high-performance Groq Cloud model inference, layout zoom constraints, and real-time incident parameter context loaders.

---

## 🚀 Key Milestones Completed

### 🤖 Phase 15A — AI Security Copilot Upgrade
* **Incident-Aware Context Routing**: Modified the FastAPI backend to query database attack event states dynamically and inject them as system context blocks (`[ATTACK EVENT CONTEXT]`) directly into LLM completion requests.
* **Structured Parameter Quick Scans**: Overhauled the Quick Scans dashboard drawer to construct custom parameters-driven prompts (incorporating IPs, ports, raw HTTP payloads, and threat score configurations) to prevent generic chatbot interactions.
* **Tokenizing Parser Overhaul**: Engineered a robust tokenizer in React to correctly parse, align, and style lists, bold tags, percentages, MITRE tags, and inline code blocks (`.inline-code-badge`) without HTML split collisions.

### ⚡ Phase 14 — Groq Cloud Migration & Viewport Stability
* **High-Speed Cloud Inference**: Moved LLM completions from local Ollama latency limits to high-speed Groq Cloud API connections (`llama-3.3-70b-versatile`).
* **Secure Key Loading**: Programmed config models to retrieve the `GROQ_API_KEY` directly from `backend/.env` without committing credentials.
* **Locked Shell Viewport**: Applied bounds to the root stylesheet grids to fully disable horizontal/vertical scrolling on the main page wrapper, ensuring layout stability at 100% zoom.

---

## 🔒 Security Notes
* **Secrets Separation**: The backend environment configuration relies on `.env` settings which are excluded in `.gitignore` by default. Under no circumstances should live keys be pushed to git repository remotes.
* **Sandbox Containment**: The Decoy Sandbox uses heuristic parses and mock scripts simulation. Avoid loading or executing real executable files inside host structures.

---

## ⚠️ Known Limitations
* **Local Python Environment Paths**: Relies on setting `PYTHONPATH=".."` or equivalent root declarations when invoking `uvicorn` backend commands manually to prevent package import errors.
* **Offline Mock Fallback**: When no API key is supplied, AI completions fallback to predefined offline rules prompts. This may limit adaptive response flexibility.
